## 三、分步详解

### 第一步：创建交接事项

**涉及组件**：`ItemForm.tsx`

1. 用户点击 `Header` 组件右上角的"新增交接事项"按钮
2. `App.tsx:21` → `handleAddItem()` 被触发，设置 `showForm = true`
3. `ItemForm` 弹窗显示，包含字段：
   - 事项标题 (title)
   - 详细描述 (description)
   - 状态 (status): 待处理/处理中/已完成/已取消
   - 优先级 (priority): 低/中/高/紧急
   - 责任人 (assignee)
   - 报告人 (reporter)
   - 截止时间 (deadline)
   - 备注 (remarks)
4. 用户填写完成后提交，调用 `handleSubmit`
5. 数据通过 `toLocalISOString` 转换为 ISO 格式后传给 `App.tsx:31` → `handleFormSubmit`
6. `App.tsx:35` → 调用 `addItem(data)` 存入 localStorage

**核心代码路径**：
- 表单组件：`src/components/ItemForm.tsx:11`
- 新增存储：`src/utils/storage.ts:14`

### 第二步：数据持久化存储

**涉及文件**：`src/utils/storage.ts`

存储使用浏览器 `localStorage`，键名为 `duty_handover_items`。

**CRUD 操作**：

| 操作 | 函数 | 说明 |
|------|------|------|
| 读取全部 | `getItems()` | `storage.ts:5` |
| 新增 | `addItem()` | `storage.ts:14`，自动生成 id、createdAt、updatedAt |
| 更新 | `updateItem()` | `storage.ts:27`，自动更新 updatedAt |
| 删除 | `deleteItem()` | `storage.ts:41` |
| 完成 | `completeItem()` | `storage.ts:49`，设置 status=completed 和 completedAt |

**数据结构示例**：
```typescript
{
  id: "1717670000000",           // Date.now() 生成
  title: "设备巡检",
  description: "对3号机组进行例行巡检",
  status: "pending",              // pending/in_progress/completed/cancelled
  priority: "high",               // low/medium/high/urgent
  assignee: "张三",
  reporter: "李四",
  deadline: "2026-06-05T23:59:00.000",
  createdAt: "2026-06-05T08:00:00.000",
  updatedAt: "2026-06-05T08:00:00.000",
  completedAt: "2026-06-05T16:30:00.000",  // 可选，完成时设置
  remarks: "补充说明..."                    // 可选
}
```

### 第三步：汇总进入班报（统计面板）

**涉及组件**：`StatsPanel.tsx`

这是系统的"班报"功能，对所有交接事项进行汇总统计。

1. `App.tsx:18` → 应用初始化时调用 `getItems()` 读取所有数据存入 `items` state
2. `App.tsx:106` → 将 `items` 传给 `<StatsPanel items={items} />`
3. `StatsPanel.tsx:9` → 组件接收数据后计算 5 项统计指标：

| 指标 | 计算逻辑 | 对应代码 |
|------|----------|----------|
| 事项总数 | 所有 items 的数量 | `StatsPanel.tsx:10` |
| 今日新增 | `isToday(item.createdAt)` 为 true 的数量 | `StatsPanel.tsx:11` |
| 已逾期 | `isOverdue(item.deadline, item.status)` 为 true 的数量 | `StatsPanel.tsx:12` |
| 待处理 | status 为 pending 或 in_progress 的数量 | `StatsPanel.tsx:13` |
| 已完成 | status 为 completed 的数量 | `StatsPanel.tsx:14` |

**逾期判断逻辑**（`src/utils/dateUtils.ts:28`）：
- 已完成或已取消的事项不视为逾期
- 截止时间早于当前时间视为逾期

---

### 第四步：进入历史记录（筛选列表）

**涉及组件**：`FilterBar.tsx` + `App.tsx`

历史记录功能通过筛选器实现，用户可按不同维度查看事项：

1. `App.tsx:78` → 计算各筛选类型的数量 `counts`
2. `App.tsx:113` → 将 `counts` 和当前筛选状态传给 `FilterBar`
3. 用户点击筛选按钮 → `setCurrentFilter()` 更新状态
4. `App.tsx:61` → `filterItems()` 根据筛选条件过滤数据：

| 筛选类型 | 过滤规则 |
|----------|----------|
| all | 返回全部 |
| today | `isToday(createdAt)` - 今日创建 |
| overdue | `isOverdue(deadline, status)` - 已逾期 |
| pending | status 为 pending 或 in_progress - 待处理 |
| completed | status 为 completed - 已完成 |

5. `App.tsx:86` → 筛选后的数据再按以下优先级排序：
   - 逾期项优先
   - 优先级从高到低（urgent > high > medium > low）
   - 创建时间新的在前

---

### 第五步：详情查看与操作

**涉及组件**：`ItemCard.tsx`

每条交接事项以卡片形式展示，相当于详情视图：

**展示信息**：
- 状态标签（颜色区分）
- 优先级标签（颜色区分）
- "今日新增"/"已逾期" 角标
- 标题、描述
- 责任人、报告人
- 截止时间、创建时间
- 备注信息

**可执行操作**：

| 操作 | 触发 | 流程 |
|------|------|------|
| 完成 | "完成"按钮 | `App.tsx:47` → `handleComplete()` → 弹出备注输入 → `completeItem(id, remarks)` 更新状态为 completed |
| 编辑 | "编辑"按钮 | `App.tsx:26` → `handleEditItem()` → 打开 ItemForm 编辑模式 → 保存时调用 `updateItem()` |
| 删除 | "删除"按钮 | `App.tsx:54` → `handleDelete()` → 确认弹窗 → `deleteItem()` 移除 |

## 五、核心业务规则总结

1. **逾期判定**：已完成/已取消的事项不会被标记为逾期
2. **排序规则**：逾期 > 高优先级 > 新创建
3. **状态流转**：待处理 → 处理中 → 已完成（或取消）
4. **完成备注**：完成事项时可追加备注，原有备注会保留，用换行分隔
5. **数据持久化**：所有操作实时写入 localStorage，刷新页面不丢失
